# Pitch Deck

## Slide 1 - Title

Liquiphy + Fluid: Low-Latency Solana Trading Intelligence Built on Jito

## Slide 2 - Problem

Solana trading and real-time analytics are latency-sensitive. Standard RPC polling and delayed signals can miss critical windows.

## Slide 3 - Solution

A Jito-oriented architecture combining low-latency intake, fast execution research, outcome tracking, and a public Fluid dashboard layer.

## Slide 4 - Jito Integration

ShredStream-style low-latency data intake, Jito low-latency transaction send / Block Engine concepts, MEV-aware infrastructure, bundles, and fast landing.

## Slide 5 - Architecture

Intake -> parser -> signal/inference layer -> policy boundary -> dry-run sender -> outcome watcher -> Fluid dashboard.

## Slide 6 - Public Demo

Selected Rust modules, Python demo data tools, fake/redacted metrics, public dashboard links, and a dry-run-only execution path.

## Slide 7 - Private Edge

Production strategy, wallet routing, insider grading, live signing, and execution policy remain private for security and IP protection.

## Slide 8 - Differentiation

Rust hot-path thinking, honest outcome tracking, public/private architecture boundary, Jito-native latency focus, and Fluid visual layer.

## Slide 9 - Ecosystem Value

Shows how Jito infrastructure can support latency-sensitive Solana applications beyond simple bots: analytics, execution research, dashboards, and safer tooling.

## Slide 10 - Team + Roadmap

Germany-based team. Next steps: stronger demo dashboard, deeper Jito integration, safer backtesting, richer public analytics, and continued private-core hardening.
