# Pitch Deck

## 1. Title

Liquiphy + Fluid: Low-Latency Solana Trading Intelligence Built on Jito

## 2. Problem

Solana trading and real-time analytics are latency-sensitive. Standard RPC polling and delayed signals can miss critical windows.

## 3. Solution

A Jito-oriented architecture combining low-latency intake, execution research, outcome tracking, and a public Fluid dashboard layer.

## 4. Jito Integration

ShredStream-style intake, Jito low-latency transaction send / Block Engine concepts, MEV-aware infrastructure, bundles, and fast landing.

## 5. Architecture

Intake -> parser -> signal layer -> policy boundary -> dry-run sender -> outcome watcher -> Fluid dashboard.

## 6. Public Demo

Public-safe Rust modules, Python demo data tools, fake/redacted metrics, dashboard links, and dry-run-only execution paths.

## 7. Private Edge

Production strategy, wallet routing, insider grading, live signing, and execution policy remain private for security and IP protection.

## 8. Differentiation

Rust hot-path thinking, outcome tracking, public/private boundary, Jito-native latency focus, and Fluid visual layer.

## 9. Ecosystem Value

Shows how Jito infrastructure can support latency-sensitive Solana applications beyond simple bots: analytics, execution research, dashboards, and safer tooling.

## 10. Roadmap

Stronger demo dashboard, deeper Jito integration, safer backtesting, richer public analytics, and continued private-core hardening.
